新增 Prisma 資料模型

// 月賽季
model Season { id String @id @default(cuid())
  key String @unique // "2026-07"
  startAt DateTime  endAt DateTime
  status SeasonStatus @default(ACTIVE) } // ACTIVE/CLOSING/CLOSED

// 月末凍結的排名快照（每季·每榜·每人）
model SeasonStanding { id String @id @default(cuid())
  seasonId String  userId String
  board HonorBoard // ACCURACY / PROFIT / INFLUENCE
  value Decimal  rank Int  settledCount Int
  @@unique([seasonId, board, userId]) }

// 成就徽章 earned 紀錄（永久）
model HonorAward { id String @id @default(cuid())
  userId String
  code String // STREAK_10 / UPSET_HUNTER / SHARP_SHOOTER / SEASON_ORACLE_1 …
  seasonId String?  value Decimal?  awardedAt DateTime @default(now())
  @@unique([userId, code, seasonId]) } // 同成就同季不重複

// 冠軍在位（月冠 → 次月在位）
model ChampionReign { id String @id @default(cuid())
  seasonId String  board HonorBoard  userId String
  reignFrom DateTime  reignTo DateTime
  @@unique([seasonId, board]) }

// 名人堂紀錄（可被超越，保留歷史）
model HallOfFameRecord { id String @id @default(cuid())
  recordType String // LONGEST_STREAK / BIGGEST_UPSET / BEST_MONTH_ACC / TOP_INFLUENCE
  userId String  value Decimal  context String?  achievedAt DateTime
  isCurrent Boolean @default(true) }

// 個人下注統計（連勝計數，維護式）
model UserBettingStat { userId String @id
  currentStreak Int @default(0)  bestStreak Int @default(0) }

// 影響力/跟單（二期新功能）
model PickFollow { id String @id @default(cuid())
  pickBetId String  followerId String  followBetId String? // 實下才算
  createdAt DateTime @default(now())
  @@unique([pickBetId, followerId]) } // 每人每單去重
  • 另加 enum HonorBoard(ACCURACY/PROFIT/INFLUENCE)、SeasonStatus。冠軍限定裝飾與各成就徽章＝CosmeticItem（purchasable=false，買不到），發放走 UserCosmetic。

防刷落點（規則 → 程式）

  • 準度/連勝 只計 odds≥1.5：沿用 WINRATE_MIN_ODDS，在 settlement hook 與 leaderboard SQL 一致 filter。
  • 最低注額門檻：settlement hook 判 stake ≥ FLOOR 才計入榮耀（FLOAT 數字待 P幣經濟定）。
  • 只計有盤口賽事：Bet 一定綁 OddsQuote，天生排除 NBA 無盤口。
  • 手機驗證才上榜/計影響力：榜/影響力查詢 join user 驗證狀態。
  • 跟單去重：PickFollow @@unique([pickBetId, followerId])＋followerId 須驗證。
⚠ Prod 注意（沿用博客邦既有教訓）
  • prod 走 db push、不走 migration：新表上 prod 後，任何 backfill（如既有用戶的 UserBettingStat 連勝回填、歷史成就補發）要手動補跑腳本，不會自動。
  • 冠軍限定裝飾靠 expiresAt 自動收回——別另寫即時卸下邏輯，避免與月 cron 打架。
  • 連勝計數是「維護式」：若 settlement 有補結算/重放，連勝重算要冪等（可由 Bet 歷史重建，不要只靠累加）。
