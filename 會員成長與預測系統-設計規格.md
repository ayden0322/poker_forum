# 會員成長與預測系統 — 設計規格（三期）

> 狀態：設計定案（2026-06-08 與 Ayden 討論確認），尚未開工。
> 本文件為「目標設計」，實作狀態以各期實作後的對帳為準，勿把本文當完成式。

---

## 0. 定位與法遵紅線（不可逆，最高優先）

- **純虛擬經濟**：G幣 / P幣 **永遠不可用真錢儲值、不可提現**，獎勵僅為虛擬裝飾（頭像框 / 勳章 / 稱號）。
- 一旦開放真錢買幣或提現，性質會從「遊戲化參與」變成「實質賭博」，台灣法律風險完全不同 → **永遠不碰真錢**。

## 1. 核心迴圈

```
每天上線
 → 做每日任務拿 G幣 + 經驗
 → 升等（吃經驗）
 → G幣 換 P幣（單向）
 → 投 P幣 預測賽事
 → 上排行榜（週/月獲利榜、勝率榜）
 → 漲粉 / 賣高手推薦
```

## 2. 雙貨幣 + 經驗

| 項目 | 取得 | 用途 | 備註 |
|---|---|---|---|
| **經驗** | 每日任務 | 只用來升等 | 永久、不可花費 |
| **G幣** | 每日任務（免費、好賺） | 裝飾商店、換 P幣 | 論壇積分 |
| **P幣** | **只能用 G幣換**（單向） | 預測下注、P幣專屬商品 | 論壇核心籌碼，**不可換回 G幣** |

- G→P 兌換：**單向**，預設 100 G幣 = 1,000 P幣（比例後台可調）。
- 贏的 P幣只能：繼續預測 + 上排行榜 + 花在 P幣 專屬商品（如預測大賽門票、限定預測勳章）。兩個經濟乾淨分離。

## 3. 架構原則：後台可調 vs 程式碼

**「有哪些旋鈕」由程式定義，「旋鈕轉到幾」由後台調。** 遊戲經濟一定會反覆調平衡，數值寫死會逼著重部署。

| 後台可調（數值 / 規則，營運自己改） | 寫在程式碼（機制 / 結構，工程才動） |
|---|---|
| 各任務獎勵 G幣 / 經驗、每日上限 | 任務「類型」（登入 / 瀏覽 / 發文 / 回覆 / 按讚＝綁程式事件） |
| 等級門檻（各級經驗值） | 升等計算邏輯、結算邏輯 |
| 各等級權益數值（G幣上限、單注上限、第幾級解鎖賣推薦） | 「權益種類」有哪些（程式先定義可掛什麼權益） |
| 商店品項與 G幣 / P幣 定價 | 防刷機制、賠率快照、結算 |
| G→P 兌換比例、預測抽水率、單注上下限 | 排行榜計算與重置排程 |
| 聯盟 × 玩法 開關（哪些賽事 / 玩法可預測） | API-Sports 盤口抓取與快照 |

## 第一期：每日任務 + 等級 + G幣 + 裝飾商店

### 每日任務
- **重置**：台灣時間 00:00。
- **規則**：各子任務**一天只能完成一次**，重刷不給獎勵；**去重計數**（瀏覽 / 按讚 / 回覆都要不同文章）。
- **無品質門檻**（一天一次即可，不限字數、不撤獎）。

| 任務 | G幣 | 經驗 |
|---|---|---|
| 每日登入 | +10 | +10 |
| 瀏覽 5 篇（不同文章） | +5 | +5 |
| 發表 1 篇文章 | +20 | +20 |
| 回覆 3 篇（不同文章） | +10 | +10 |
| 按讚 5 次（不同文章） | +5 | +5 |
| **每日上限** | **50** | **50** |

### 等級系統（吃經驗）

| 等級 | 名稱 | 經驗門檻 |
|---|---|---|
| Lv1 | 新手 | 0 ~ 999 |
| Lv2 | 球探 | 1,000 ~ 2,999 |
| Lv3 | 分析師 | 3,000 ~ 9,999 |
| Lv4 | 專家 | 10,000+ |
| Lv5 | 名人堂 | 邀請制（手動） |

- **綁實際權益**（權益數值後台可調）：每日 G幣上限、單注 P幣上限（二期）、Lv3+ 才能賣推薦（三期）、各級專屬勳章 / 頭像框。
- 升等曲線備註：Lv4 需 10,000 經驗 ÷ 每日最多 50 = 最少約 200 天。

### G幣 用途（第一期）
- **虛擬裝飾商店**：頭像框 / 勳章 / 稱號（純虛擬，後台可上架與定價）。

## 第二期：G→P 兌換 + 預測 + 排行榜

### 預測系統

- **賠率來源**：**全部來自 API-Sports**（聚合各家盤口），**系統不自行計算賠率**。
  流程：列賽事抓當前盤口 → **投注瞬間快照鎖定賠率/盤口線** → 賽後拿最終比分對結果結算。
- **玩法**：
  - **勝負**：投注額 × 該隊賠率。
    例：投 500 P幣 壓湖人 @1.85 → 贏拿回 925（淨 +425）；輸失去 500。
  - **大小分**：總分 vs 盤口線。
    例：盤口線 220.5、大 @1.90 → 投 500 壓大；總分 228 > 220.5 → 拿回 950；總分 215 < 220.5 → 輸 500。
    （整數盤口線且總分剛好等於 → 平盤，**退回本金**。）
- **鎖單**：開賽前自動關閉投注。
- **延賽 / 腰斬 / 取消**：全額**退回 P幣**。
- **資料源開關**：依 API-Sports 盤口覆蓋度，**聯盟 × 玩法**後台開關
  （MLB / NBA 通常有大小分；中職 / 日韓職可能只開勝負；完全無盤口的賽事不開放預測）。
- **單注上下限**：後台可調、可綁等級。

### 排行榜（台灣時區）

| 榜種 | 內容 | 重置 |
|---|---|---|
| 本週獲利榜 | 期間內淨 P幣 損益（贏 - 輸） | 週一 00:00 |
| 本月獲利榜 | 同上 | 每月 1 號 00:00 |
| 勝率榜 | 勝場 / 已結算場；**滿 30 場**才列入 | （持續累積） |

- **獎勵**：前三名 = **限定稱號掛一週**（如「本週預測王」框，掛到下週揭曉）+ **永久紀念勳章**（收藏品）。
- 純虛擬仍要**防多帳號刷榜**（榜單榮譽會被洗）。

## 第三期：創作者榮譽（定案改向 2026-07-07 圓桌）

> **定案：走「榮譽版」，不開放會員付費買推薦。**
> 原「賣預測推薦」構想經圓桌評估否決，理由：帳號會產生場外真錢交易價值（黑市買賣號管不到）、
> 榜單等於替賣料仔做信任背書、站點在 Google 政策眼中從運動社群滑向博彩服務（賭上 SEO 命脈）。

- 高手（Lv3+）可發**賽前預測貼**、上首頁精選位；獎勵由**平台發放**（G幣 / 限定稱號），不開 user-to-user 付費。
- 漲粉與預測戰績連動（追蹤、戰績頁）照做——成長價值不需要「能賣錢」才成立。
- 二期的戰績頁**不預留**訂閱/付費版位（不留鉤子）。

## 實作時待補確認（細節）

- 平手處理：若未來納入會平手的賽事（足球勝負），需定義平盤退本金或設第三選項。
- API-Sports 多家盤口：要選定主要 bookmaker 或取平均。
- 反作弊：多帳號 / 同裝置 / IP 的偵測強度（純虛擬但榜單公信力需要）。
- 既有基礎：DB 已有 `users.level`、`follows`、`pushes`（按讚）可作為等級 / 漲粉 / 任務的地基。

## 已確認決策清單（討論於 2026-06-08）

1. 預測賠率模型 = **真實賠率**（API-Sports）
2. 開發分期 = **三期**（任務 → 預測 → 創作經濟）
3. 法遵 = **永遠純虛擬**（不儲值、不提現）
4. 等級 = **綁實際權益**，且權益**數值後台可設定**
5. 第一期 G幣 去處 = **虛擬裝飾商店**
6. 防刷 = **各子任務一天一次**，重刷不給（無品質門檻）
7. P幣 = **單向 G→P，不可回換**
8. 預測玩法 = **勝負 + 大小分**（賠率全取自 API-Sports，不自算）
9. 排行榜獎勵 = **限定稱號掛一週 + 永久勳章**
10. 結算機制 = 投注當下鎖賠率、開賽前鎖單、延賽退幣、聯盟 × 玩法後台開關

## 已確認決策清單（圓桌討論於 2026-07-07）
