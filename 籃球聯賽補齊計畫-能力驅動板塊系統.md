## 0. 背景與業主需求

- 博弈論壇 SEO 站（`goboka.net`）籃球目前只完成 **NBA** 板塊。
- **業主要求**：博弈板塊資訊盡可能齊全，每個整理好的聯賽要**獨立成板塊**（不丟進「其他籃球」）。
- 範圍決策（2026-06-10 拍板）：**全開 Tier A+B 主要聯賽**；Tier C（薄資料）先查官方源再決定；每個新聯賽動工前先花 10 分鐘查官方免費源。
- **資料策略（2026-06-10 拍板）：以「資訊完整」為最高導向，API-Sports 配額不用省**。→ 所有有賽程的聯賽預設開 LIVE 即時輪詢；能拿的數據都拿、能開的 widget 都開。（唯一仍須處理的是 key 6/14 到期，那是「會不會 401」的存活問題，不是配額問題。）

## 1. API-Sports 籃球能力盤點（2026-06-10 實測）

- 帳號 Pro 7500/日，**⚠️ 2026-06-14 到期**（正式站 friendlies LIVE cron 也吃同把 key，須續訂，與本專案解耦）。
- 籃球 base：`v1.basketball.api-sports.io`。共 427 聯賽、298 有現役賽季。
- 每聯賽 coverage 逐一不同（standings / players名單 / 球隊比賽統計 / 球員比賽統計 / 賠率），**不可假設一致**（棒球 `/players` 死碼覆轍）。
- **關鍵**：`/games` 端點即使 coverage 的 statistics=false，仍回賽程+**比賽結果(含分節比分)**；statistics 端點才是 box score。實測 PBA(151) 89 場全有結果，但 box score / 球員統計 results=0。

## 2. 資料源策略（三套 adapter）

| 來源 | 適用 | 說明 |
|---|---|---|
| ESPN 免費 | NBA（已完成） | `site.api.espn.com` + `cdn.nba.com` + `stats.nba.com`，免費、3 秒輪詢直播。ESPN 籃球只蓋 NBA/WNBA/NBL澳洲/NCAA，**亞洲聯賽全不蓋、EuroLeague 也不蓋**。 |
| 官方免費 API | TPBL（已驗證） | `api.tpbl.basketball/api`，無需 key，資料比 API-Sports 完整（連投籃熱區都有）。詳見 memory `reference_tpbl_free_api.md`。 |

**原則**：每接一個新聯賽，動工前先查官方/第三方免費源（紅線 `feedback_free_api_first.md`）。對 API-Sports 服務差的聯賽（PBA 無球員數據）尤其值得查。

## 3. 能力驅動架構（解決「業主要盡量多」的核心）

不為每個聯賽手刻。建**一套通用籃球板塊系統**：

- `LEAGUE_CONFIG` 每聯賽標注 `capabilities`（**源無關的能力宣告**，非綁 API-Sports flag）：
  ```ts
  capabilities: {
    standings: boolean;  // 排行榜
    schedule:  boolean;  // 賽程 + 比賽結果(比分)
    boxScore:  boolean;  // 單場球隊/球員統計
    players:   boolean;  // 球員名單 / 球員頁
    odds:      boolean;  // 賠率（競猜用）
    live:      boolean;  // 即時輪詢
  }
  dataSource: 'espn' | 'apisports' | 'tpbl-official' | 'pba-official';
  ```
- 通用 widget **按 capabilities 決定渲染什麼**：有 odds 出競猜卡、有 players 出數據王、只有 standings+schedule 就只出排名+賽程結果（不出空的球員頁）。
- 開新聯賽 = 加一筆 config + board record + 翻譯 seed，**不寫新程式**。每個都是獨立 slug / 獨立板塊 / 獨立詳情頁。
- ⚠️ capabilities 會逐季變，註記「資料截至 2026-06、需定期核對」，別變成新死碼。

## 5. SEO 要求（不可省）

1. **空殼板塊先 noindex**：未上數據的板塊一律 noindex（已實作 `apps/web/src/lib/board-seo.ts` + `board/[slug]/page.tsx` + `sitemap.ts`）。**補一個聯賽上線就從黑名單刪一行**解除 noindex。
2. **賽程/排行頁 SSR**：數據必須出在 HTML，client fetch 爬蟲看不到 → 長尾進不了索引。
3. **比賽詳情頁分級索引**：固定球隊一季上百場高重複，例行賽 box score 用 `noindex,follow`；焦點戰/有討論量才開索引。**勿照抄友誼賽「全索引」決策**（那是國際隊低重複場景）。
4. **結構化資料**：`SportsEvent`（賽事）、`SportsTeam`（球隊）、`ItemList`（賽程）。
5. **inline CTA**：每場賽程旁放「聊這場 / 競猜這場」，接在「查完賽程的動作」後，非頁尾通用 CTA。
