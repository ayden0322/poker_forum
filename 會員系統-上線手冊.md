## 1. 不變量（會紅燈的測試守著，不靠人記）

`apps/api/src/economy/economy-invariants.spec.ts` 守三條鐵律，改 economy 打破其一就紅燈：

1. **法遵紅線**：`Currency` 只有 G/P/EXP、`LedgerReason` 無任何真錢儲值/提現概念（純虛擬經濟，台灣法律紅線）。新增帳本理由＝必須有意識改這個測試。
2. **防刷**：瀏覽任務必須驗證文章存在（擋捏造 id 刷幣）；同 refId 當日去重。
3. **fail-closed**：開關關閉時 `recordEvent` no-op、member 讀取 API 回 `enabled:false` 不洩資料。

跑法：`docker compose run --rm api sh -c "cd apps/api && pnpm test"`（目前 37 綠）。

## 2. Go-Live 安全順序（⚠️ 有不可逆步驟，務必照順序）

> 核心原則：**fail-closed 開關保護「翻開前」，保護不了「翻開那一刻」**。
> 「重設舊會員 level」是不可逆的全表 UPDATE，必須在翻開關**之前**、且備份+驗證過才做。
> 翻開關是可逆的（翻回去即可），所以放最後。

### 階段 A：上程式（行為仍關，零風險）
1. 確認 `feat/economy-4-rewire` 的 prisma migration 與部署線 DB 對得上（schema 已含 `task_event_log`、`daily_task_defs.threshold` 等）。
2. 合併分支 → 部署。此時 `MEMBER_ECONOMY_ENABLED` 仍**不設（即關）**，前台無任何變化。
3. 煙霧測試：登入/發文/按讚一切如常（會員 UI 不出現、不發幣）。

### 階段 B：資料對齊（不可逆，先備份）
4. **備份 `users` 表**（至少 `id, level`），例：
   `pg_dump -t users ... > users_backup_YYYYMMDD.sql`
5. **跑 level 重設 dry-run**（不寫入），檢視變更分佈是否合理：
   `docker compose exec api sh -c "cd packages/database && pnpm reset-member-levels"`
   - 預期：舊發文數時代的 Lv5/Lv6（經驗多為 0）→ 降回 Lv1，避免被誤判成「邀請制名人堂」。
6. 確認 dry-run 報告無誤後，**對 prod 跑 `--apply`**（已備份才做）：
   `... pnpm tsx prisma/reset-member-levels.ts --apply`
7. 抽查幾個會員 level 是否正確。
   > ⚠️ 此腳本會依經驗重算**所有**會員。go-live 當下尚無「真正的邀請制名人堂(Lv5)」會員，故安全；未來若已有邀請會員，**勿無腦重跑**（會把他們降級）。

### 階段 C：翻開關（可逆，放最後）
8. 二期預測上線前置（若同時）：**賠率查詢必須走 Redis 快取**，不可讓前端直打 API-Sports（會員數×賽事數流量，會炸額度）。一期不涉及。
9. Zeabur 設 `MEMBER_ECONOMY_ENABLED=true` → redeploy。
10. 驗證：登入→G/EXP+10；瀏覽5篇→+5；Header 出現徽章；會員中心正確。

### 回滾
- 行為層：把 `MEMBER_ECONOMY_ENABLED` 設回 false（或移除）→ redeploy，前台立即恢復無會員系統。
- 資料層：level 重設不可逆 → 這就是階段 B 必須先備份+dry-run 驗證的原因；真要救只能還原 `users` 備份。

## 3. 關鍵決策記錄（ADR 摘要）

| 決策 | 內容 / 理由 |
|---|---|
| 純虛擬經濟 | G/P 幣不可儲值真錢、不可提現（台灣法律紅線）。以不變量測試守。 |
| fail-closed 開關 | schema 先上、行為預設關，go-live 只翻一個環境變數；但不可逆資料遷移不靠它。 |
| #4 孤兒 commit 重接 | 事件接線 #4 曾卡在停滯分支從沒進部署線；以 cherry-pick 乾淨重接（schema 已驗對齊）。 |
| 瀏覽任務改前端回報 | 文章頁是 SSR 匿名抓取、拿不到 token；改 `POST /posts/:id/view` 由前端登入後回報，後端驗證文章存在 + 去重防刷。 |
| token 自動刷新 | access token 15m，帶 token 的請求過期也要自動刷新重試 + 同步 context，只用本 session token。 |
| 代登入 refresh | refresh 須保留 `impersonatedBy` 並維持 1h 短效，不可升級成 7d 正常 token（否則代登入變永久身分盜用）。 |
| 二期賠率 | 預測賠率取自 API-Sports，上線前必須走 Redis 快取層。 |
| 瀏覽器驗證交給 agent | 真實使用者路徑測試用 headless 瀏覽器 agent 跑，不佔用真人瀏覽器；禁止只用 API/token 注入當通過證據。 |

## 4. 常用驗證指令

```bash
# 不變量 + 規則引擎測試（37 綠）
docker compose run --rm api sh -c "cd apps/api && pnpm test"

# API 型別/建置
docker compose exec api sh -c "cd apps/api && pnpm build"

# 前端型別 / 真正的 prod build（build 後記得清 .next + 重啟 web，避免弄壞 dev 樣式）
docker compose exec web sh -c "cd apps/web && npx tsc --noEmit"
docker compose exec web sh -c "cd apps/web && pnpm build"

# level 重設 dry-run / 實際
docker compose exec api sh -c "cd packages/database && pnpm reset-member-levels"
docker compose exec api sh -c "cd packages/database && pnpm tsx prisma/reset-member-levels.ts --apply"
```
